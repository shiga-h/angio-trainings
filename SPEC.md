# アンギオ部門 研修会・学会一覧サイト 仕様書

Claude Codeへの引き継ぎ用。この仕様通りに実装し、Vercelへデプロイしてください。

## 目的

アンギオ部門スタッフがスマホから研修会・学会の情報を見られるようにする。
先輩が配布するExcelをデータの源とし、その閲覧用レイヤーをWebサイトとして作る。

## データソース

添付の `2026年度研修会_統合版.xlsx` を読み込む。シート名は「研究会一覧」。

列構成（1行目がヘッダー）:

| 列 | 内容 |
|---|---|
| ID | 例: 2026-001 |
| 開始日 | YYYY-MM-DD |
| 終了日 | YYYY-MM-DD（単日開催は空欄） |
| 時間 | 例: 15:00〜18:00（空欄可） |
| 名称 | 研究会名 |
| テーマ | 空欄可 |
| 開催場所 | |
| 開催形式 | web / 現地 / Hybrid / オンデマンド / 未定 |
| 参加費 | |
| 認定単位 | 空欄可、複数行の場合あり |
| HP | URL、空欄の行もある |
| タグ1 | 下記8タグのいずれか |
| タグ2 | 下記8タグのいずれか、空欄可（タグなしの行もある） |

名称が空の行（IDのみ振られた予備行）はスキップする。

## タグ体系（8種、固定）

循環器／脳神経／消化器／その他アンギオ／画像診断／線量管理／画像管理／その他モダリティ

タグ1・タグ2を配列にまとめて扱う（例: `["循環器", "線量管理"]`）。

## 必須機能

- **月で絞り込み**（データに存在する月のみチップ表示、「すべて」も用意）
- **タグで絞り込み**（複数選択可、OR条件）
- **キーワード検索**（名称・テーマを対象）
- **終了済みの非表示をデフォルトにする**（開始日が今日より前のものは隠す。終了日がある場合は終了日基準。日付が「未定」（空欄）の行は終了済み判定せず常に表示する）
- 「過去の研究会も表示」を切り替えるボタン/トグルを設置する

## 不要な機能（実装しない）

- お気に入り登録機能（MVPから除外する判断済み）
- ログイン・認証（社内向けの簡易公開のため不要）

## 表示項目（1件分のカード）

日付（開始日、複数日なら「開始日〜終了日」表記）、時間、名称、テーマ、開催場所、開催形式（バッジ）、タグ（バッジ、複数）、参加費、HPリンク（urlが空の場合はリンクを表示しない）。
認定単位は長くなる場合があるので、カード内で折りたたみ表示にするか、省略して詳細はHPリンク先に委ねる形でよい。

## デザイン方針

院内スタッフ向けの実用ツール。派手な装飾は不要、視認性・読み込み速度・スマホでの操作性を優先する。
モバイル優先（iPhoneでの利用が主、MacBook/iPadでも見られること）。
外部CDN依存は最小限にし、シンプルなHTML/CSS/JSで完結させる（オフラインや院内ネットワークの制約を考慮）。

## ホスティング・運用

- GitHubリポジトリを作成し、Vercelと連携済み（GitHub連携によるVercelアカウントが既にある）の状態を前提とする
- リポジトリにpushすれば自動的にVercelが再デプロイする構成にする（Vercelの「GitHub連携」機能を使う。CLIでの手動デプロイは不要）
- 更新フローは不定期: Excelを更新 → 変換スクリプトでdata.json再生成 → git commit → git push（これで自動的に本番反映される）
- 変換スクリプトの骨格は `convert.py` として既にある（openpyxlでExcelを読み込み、JSON出力する。引き継いで構わない）

## 確定済みの判断（変更不要、参考情報）

- 月別シートではなく1枚のシートに統合済み（理由: データベースとしての扱いやすさ優先）
- 先輩のExcel配布方法（メール添付）は変更しない。志賀さんが原本シートを別途保持し、差分を手動で転記する運用
- IDは将来の拡張（お気に入り機能等）に備えた保険。今は使わなくてよい
